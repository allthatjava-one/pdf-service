# Requirements
- [ ] **Background Remover:** The agent must be able to remove the background from an image, leaving only the foreground subject intact. This functionality should be accessible through an API endpoint that accepts an image file and returns the processed image with the background removed.
- [ ] **New End Point:** The agent must provide a new API endpoint `/remove-background` that accepts an image file and returns the processed image with the background removed. This endpoint should be added in AGENTS.md and implemented in the backend service.
- [ ] **Request body:** The request body for the `/remove-background` endpoint should include R2 objectKey. 
The feature should take the Cloudflare R2 Storage objectId then process the file and return the result(presignedUrl) to api-gateway, then api-gateway will send the result to the agent.
- [ ] **Code location:** The code for the background remover should be implemented in a new module within the backend service, following the existing code structure and conventions. The new API endpoint should be added to the appropriate router in the FastAPI application. The physical python code should be placed user `src/` directory and named `background_remover.py`.
- [ ] **Testing:** The agent must include appropriate tests to ensure the functionality of the background remover. This should include unit tests for the image processing function and integration tests for the new API endpoint. All tests should be placed in the `tests/` directory and follow the naming convention `test_*.py`.

## Additional requirements
- [ ] **Use tech:** should use rembg as the background remover library, which is a popular open-source tool for removing backgrounds from images. The implementation should leverage the capabilities of rembg to achieve accurate and efficient background removal.
- [ ] **Documentation:** The agent must update the documentation to include information about the new background

## Additional requirements 2
- [ ] **New request parameter**: The agent must accept a new request parameter `quality` in the `/remove-background` endpoint. This parameter should allow users to specify the desired quality of the output image, with possible values such as `low`, `medium`, and `high`. The agent should adjust the processing accordingly to produce images of varying quality based on the user's selection.
- [ ] **New request parameter**: The agent must accept a new request parameter `type` in the `/remove-background` endpoint. This parameter should allow users to specify the desired output image format, such as `remove` or `blur`. If the type is `remove`, the agent should produce an image with the background completely removed (transparent). If the type is `blur`, the agent should produce an image where the background is blurred instead of removed, while keeping the foreground subject intact. The agent should process the image accordingly based on the specified type.
- [ ] **New optional request parameter**: The agent must accept a new optional request parameter `blur-strength` as `light`, `medium`, or `strong` in the `/remove-background` endpoint. This parameter should allow users to specify a threshold value for background blur when the `type` parameter is set to `blur`. The agent should adjust the intensity of the background blur based on the specified `blur-strength`, providing users with control over the level of blurring applied to the background while keeping the foreground subject clear.
- [ ] **Image type**: If the request parameter `type` is set to `blur`, converted image should be in same formate as the input image. If the request parameter `type` is set to `remove`, converted image should be in PNG format with transparent background, regardless of the input image format.
- [ ] **Add qaulity control**: add following control for output image quality with technique of rembg:
`final = final.filter(ImageFilter.SHARPEN) if quality == "high" else final.filter(ImageFilter.SMOOTH) if quality == "low" else final`
- [ ] **Renamed the file**: If `type` is `remove`, the output file should be named with suffix `_removed.png`. If `type` is `blur`, the output file should be named with suffix `_blurred` and preserve the original file extension (e.g., `.jpg`, `.jpeg`, `.png`).
